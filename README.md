## 💡 Идея проекта

### Почему Юнг?

Классические боты для толкования снов работают как справочники символов: пользователь вводит образ — бот выдаёт значение. Это поверхностно и лишено индивидуальности.

Юнгианский подход иначе:
- **Символы индивидуальны**. Тот же образ (вода, змея, дом) имеет разное значение для разных людей в разных жизненных контекстах.
- **Сновидение — диалог с бессознательным**. Аналитик не «расшифровывает» сон, а исследует его вместе с сновидцем.
- **Накопление смысла**. С течением времени выявляются повторяющиеся архетипы, эмоциональные паттерны и теневые аспекты личности.

### Что делает бот уникальным?

1. **Персонаж с характером** — не бездушная машина, а «Доктор Юнг»: любопытный, эмпатичный, философский, никогда клинический и холодный.
2. **Память о пользователе** — бот помнит предыдущие сны, повторяющиеся символы и эмоциональные паттерны, ссылается на них в новых анализах.
3. **Голосовой ввод** — можно рассказать сон голосом сразу после пробуждения, не вставая с кровати.
4. **Визуализация** — каждый сон иллюстрируется сюрреалистичным изображением через Pollinations AI.
5. **Утренние напоминания** — бот сам напоминает записать свежий сон, пока воспоминания не выветрились.
6. **Следующий вопрос** — после каждого анализа Доктор задаёт один глубокий вопрос о символе или эмоции, развивая диалог.

## 🏗 Архитектура и логика

### Поток данных

```
Пользователь
    │
    ├─→ Текст ─────────────────┐
    ├─→ Голосовое сообщение ───┤→ Whisper (Groq) → Текст
    └─→ Аудиофайл ─────────────┘
                                    │
                                    ▼
                    ┌───────────────────────────────┐
                    │      Юнгианский анализ        │
                    │    (Groq LLM + System Prompt) │
                    └───────────────────────────────┘
                                    │
                    ├─→ Ответ пользователю (Markdown)
                    ├─→ Обновление психологического профиля
                    ├─→ Сохранение в журнал
                    └─→ Генерация изображения (Pollinations AI)
```

### Состояния диалога

Бот использует конечный автомат с двумя состояниями:

| Состояние | Описание | Переход |
|-----------|----------|---------|
| `idle` | Ожидание нового сна | Пользователь отправляет сон → `followup` |
| `followup` | Ожидание ответа на вопрос Доктора | Пользователь отвечает → `idle` |

**Почему это важно:** в состоянии `followup` бот не интерпретирует сообщение как новый сон, а использует его как ответ на предыдущий вопрос, что позволяет углубить анализ и обогатить психологический профиль.

### Психологический профиль

Для каждого пользователя бот поддерживает структурированный JSON-профиль:

```json
{
  "archetypes":         ["Тень", "Анима", "Мудрый старец"],
  "recurring_symbols":  ["вода", "падение", "неизвестный дом"],
  "emotional_patterns": ["тревога перед важным событием", "освобождение после конфликта"],
  "fears":              ["потеря контроля", "неизвестность"],
  "themes":             ["путешествие", "преодоление препятствий"],
  "last_updated":       "2024-05-12"
}
```

**Как обновляется:** после каждой сессии (сон + анализ + возможный ответ на вопрос) Groq получает текущий профиль и новые данные, затем возвращает обновлённую версию. Старые данные не затираются — новые дополняются (максимум 7 элементов в каждом списке).

**Как используется:** профиль инжектируется в System Prompt перед каждым анализом, поэтому Доктор может сказать: *«Вода появляется в ваших снах в третий раз — в прошлый раз это было связано с…»*

## 📁 Структура кода

```
Bot_dream.py
├── Импорты и конфигурация          # Токены, константы, инициализация клиентов
├── SYSTEM_PROMPT                   # Персона Доктора Юнга + правила анализа
├── In-memory state                 # История диалогов, состояния пользователей
├── APScheduler                     # Фоновый планировщик напоминаний
│
├── Модуль Journal                  # Загрузка/сохранение журнала снов
├── Модуль Psychological Profile    # Загрузка/обновление/форматирование профиля
├── Модуль Image Generation         # Генерация промпта + скачивание картинки
├── Модуль Audio Transcription      # Whisper: голос → текст
├── Модуль Reminders                # Планирование/восстановление напоминаний
│
├── Core Analysis                   # Главная логика анализа сна
├── Handle Followup                 # Обработка ответа на вопрос Доктора
│
├── Command Handlers                # /start, /profile, /remind, /journal и др.
├── Text Handler                    # Маршрутизация текстовых сообщений
├── Voice/Audio Handlers            # Обработка голосовых и аудио
│
└── Entry Point                     # Восстановление напоминаний + polling
```

### Ключевые функции

| Функция | Назначение |
|---------|------------|
| `run_analysis()` | Главный конвейер: анализ сна, сохранение, генерация картинки, обновление профиля |
| `handle_followup_answer()` | Углублённый анализ после ответа пользователя на вопрос Доктора |
| `update_profile()` | LLM-агент, который мёржит новые данные в психологический профиль |
| `transcribe_audio()` | Расшифровка голоса через Whisper Large v3 |
| `generate_image_prompt()` | LLM создаёт короткий визуальный промпт для генерации картинки |
| `fetch_dream_image()` | Скачивание изображения с Pollinations AI |
| `schedule_reminder()` | Cron-напоминание на конкретное время |

## 🤖 Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие, сброс сессии, список команд |
| `/reset` | Псевдоним `/start` — сброс истории диалога |
| `/profile` | Показать накопленный юнгианский профиль |
| `/history` | Сводка текущей сессии + юнгианская интерпретация всех снов |
| `/journal` | Последние 10 сохранённых снов с датами |
| `/clearjournal` | Полная очистка журнала |
| `/remind HH:MM` | Установить ежедневное напоминание (например, `/remind 07:30`) |
| `/remind off` | Отключить напоминание |

## 🛠 Технологический стек

| Компонент | Технология | Роль |
|-----------|-----------|------|
| **Telegram API** | `pyTelegramBotAPI` | Приём сообщений, отправка ответов, скачивание файлов |
| **LLM** | Groq Cloud (`llama-3.1-8b-instant`) | Юнгианский анализ, обновление профиля, генерация промптов |
| **Speech-to-Text** | Whisper Large v3 (Groq) | Расшифровка голосовых сообщений |
| **Image Generation** | Pollinations AI | Сюрреалистичные иллюстрации снов |
| **Scheduler** | APScheduler | Фоновые cron-напоминания |
| **Storage** | JSON-файлы | Журнал, профили, напоминания (stateless, zero-config) |

## 🚀 Установка и запуск

### 1. Клонирование и зависимости

```bash
git clone <repo-url>
cd bot_dream_interpretation

# Создать виртуальное окружение
python -m venv venv

# Активировать
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# Установить зависимости
pip install pyTelegramBotAPI requests groq apscheduler python-dotenv
```

### 2. Конфигурация

Создайте файл `.env` в корне проекта:

```env
TELEGRAM_TOKEN=your_telegram_bot_token_from_botfather
GROQ_API_KEY=your_groq_api_key_from_console.groq.com
```

### 3. Запуск

```bash
python Bot_dream.py
```

Вывод в терминале:
```
🌙 Restoring reminders...
  Restored reminder for 123456789 at 07:30
🌙 Dr. Jung is ready.
```

## ⚙ Конфигурация

### Переменные в коде

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| `GROQ_MODEL` | `llama-3.1-8b-instant` | Модель для всех текстовых запросов |
| `MAX_HISTORY` | `20` | Максимум сообщений в контексте диалога |
| `JOURNAL_FILE` | `journal.json` | Файл журнала снов |
| `PROFILES_FILE` | `profiles.json` | Файл психологических профилей |
| `REMINDERS_FILE` | `reminders.json` | Файл напоминаний |
| `timezone` | `Asia/Bishkek` | Часовой пояс для планировщика |

### Смена модели

Если Groq отключает текущую модель, просто измените:

```python
GROQ_MODEL = "llama-3.3-70b-versatile"  # более умная, но дороже
```

## 📂 Файлы данных

Все данные хранятся в JSON-файлах в корне проекта:

| Файл | Структура | Содержимое |
|------|-----------|------------|
| `journal.json` | `{chat_id: [{date, dream, analysis}, ...]}` | Все сны всех пользователей |
| `profiles.json` | `{chat_id: {archetypes, symbols, ...}}` | Психологические профили |
| `reminders.json` | `{chat_id: "HH:MM"}` | Время напоминаний |

> **Примечание:** хранение в JSON удобно для прототипа, но для production рекомендуется перейти на SQLite или PostgreSQL.

## 🔒 Безопасность

### Что уже сделано
- Токены вынесены в `.env` (не попадают в Git при правильном `.gitignore`)
- `.gitignore` должен содержать: `venv/`, `.env`, `*.json`

### Что нужно сделать
1. **Отозвать старые токены**, если они были скомпрометированы (попали в скриншоты, GitHub и т.д.)
2. **Создать новые** через @BotFather и console.groq.com
3. **Никогда не коммитить `.env`** — проверьте `git status` перед первым коммитом
